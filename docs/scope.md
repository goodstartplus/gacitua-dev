# gacitua.dev — Project Scope

## Vision

The goal of `gacitua.dev` is not to create a traditional resume website or a job-seeking portfolio.

This project exists to present André Gacitua as a **builder**: someone who creates real digital products, platforms, apps, websites, automations, and internal systems across different stacks, tools, and environments.

The website should communicate practical experience, creative range, technical depth, and real-world execution.

It should work as a public-facing digital profile that helps friends, clients, partners, and other professionals quickly understand:

- what André has built
- what kinds of problems he solves
- what technologies and ecosystems he has worked with
- how broad and practical his background really is

The site should feel more like a **living profile of a product-oriented developer** than a corporate portfolio or curriculum vitae.

---

## Core Positioning

This is **not** a “hire me” website.

It is a personal/professional showcase designed to document and present:

- real projects
- technical background
- product thinking
- experimentation
- practical execution across multiple kinds of work

The message behind the site should be something close to:

> I build real digital products and systems.  
> Apps, platforms, websites, automations, and ideas brought into execution.

The site should reinforce the identity of someone who is not only technical, but also deeply involved in product decisions, implementation details, infrastructure choices, iteration, and launch.

---

## Main Concept

The homepage should be heavily inspired by the structure and visual logic of an Instagram profile, but reinterpreted in a more premium, monochromatic, and professional way.

The idea is **not** to clone Instagram literally, but to borrow some of its strongest structural patterns:

- profile at the top
- personal identity first
- visual project grid
- discoverable content
- quick scanning
- project-based navigation

This concept fits the purpose of the site because André has multiple real projects, and a profile-like interface helps communicate the idea of an active creator with a growing body of work.

---

## Design Direction

### General direction
The visual identity should feel:

- monochromatic or near-monochromatic
- clean
- modern
- intentional
- minimal, but not empty
- personal, but not casual
- inspired by social product interfaces, but elevated

### Avoid
The site should avoid feeling like:

- a corporate consulting website
- a traditional CV/resume template
- a Behance-like creative portfolio
- a literal Instagram copy
- a noisy startup landing page

### Desired feeling
The website should feel like:

- a personal profile for a serious product builder
- a curated archive of real work
- a developer/product creator presence on the web
- a modern digital identity

---

## Language Strategy

The website should support **Portuguese and English**.

The content will be available in both languages through a translation system or bilingual content structure.

This is important because the site should serve both:

- Brazilian audiences (friends, clients, local partners)
- international audiences (developers, founders, potential collaborators, global visitors)

The language system should be designed from the start as part of the information architecture, not as an afterthought.

---

## Primary Goals

### 1. Showcase real experience
The site should clearly present André’s involvement in multiple real-world projects across different formats and stacks.

### 2. Create a strong personal/professional digital presence
The site should act as a central public hub representing André’s work and background.

### 3. Organize a growing portfolio of products and systems
The site should scale well as more projects are added over time.

### 4. Make exploration visual and intuitive
The visitor should be able to browse projects in a visually engaging way, with lightweight discovery similar to social media interfaces.

### 5. Register technical breadth
The site should help document the many tools, ecosystems, and technologies André has used over time.

---

## Audience

The website is primarily for:

- clients
- friends
- collaborators
- other developers
- founders
- people curious about André’s projects and technical background

It is not optimized around recruiters or hiring managers, although it should still communicate strong professional credibility.

---

## Homepage Structure

## 1. Profile Header

The top of the homepage should behave like a profile section inspired by Instagram.

It should include:

- profile photo
- name
- short title or descriptor
- short bio
- optional quick stats
- action buttons

### Purpose
This section should establish identity immediately and frame the whole site as a personal profile.

### Tone
The bio should be concise, direct, and focused on creation/building, not self-promotion in a generic way.

Example direction:

- builder / developer / product creator
- builds apps, platforms, websites, automations
- practical execution across real projects

---

## 2. Quick Stats / Profile Meta

A small line of profile-style metadata can help reinforce depth and give structure to the page.

Examples of possible metrics:

- projects built
- stacks/tools used
- categories worked on
- years building
- products launched

These do not need to be exaggerated or vanity metrics. Their role is to provide quick context and strengthen the profile feel.

---

## 3. Highlight Navigation

Inspired by Instagram story highlights, the homepage can include circular or pill-shaped highlight items that represent key categories or major projects.

Possible examples:

- Goodstart+
- Sideout
- Websites
- Automations
- Mobile
- Platforms
- Education
- Experiments
- Stack

### Purpose
This creates a strong visual bridge with the inspiration while also improving navigation.

### Behavior
Highlights may link to:

- filtered project views
- anchor sections
- category pages
- featured project pages

---

## 4. Project Grid

This is the core of the homepage.

The main content area should be a visual grid strongly inspired by Instagram’s post grid.

Each tile should represent a real project and should display:

- project image or screenshot
- optional overlay or visible tag
- project type tag such as:
  - App
  - Website
  - Platform
  - Automation
  - Internal Tool
  - Experiment

### Important note
The grid should feel visually close to an Instagram-style layout, but with a more refined and monochromatic design language.

The goal is to make browsing feel familiar, fast, and visual.

---

## 5. Project Modal

When the user clicks a project tile, the first interaction should open a modal.

This modal should present a quick overview of the project before sending the visitor to a deeper page.

### The modal should include:
- project name
- project type/category
- short description
- optional stack summary
- optional status
- optional screenshot/gallery preview
- two main actions:
  1. **Visit Site**
  2. **See More Details**

### Purpose
The modal creates a smoother browsing experience and allows the homepage to remain highly visual without forcing immediate full-page navigation.

### Buttons
- **Visit Site** → external project URL, if available
- **See More Details** → internal project detail page inside `gacitua.dev`

---

## 6. Project Detail Pages

Each project should have its own complete internal page.

These pages should serve as case-study-style detail views.

### Suggested content structure
- project title
- category/type
- overview
- context
- problem or need
- what was built
- André’s role
- technologies involved
- interesting challenges
- outcomes / current state
- screenshots or visuals
- links

### Purpose
The homepage attracts attention and encourages discovery.  
The detail page provides proof, depth, and narrative.

---

## Content Strategy for Projects

Projects should not be presented as generic portfolio items.

Each project should communicate:

- what it is
- why it exists
- what kind of system or product it became
- what kind of work was involved
- where the technical complexity or product thinking was

This matters because André’s value is not just “knowing tools,” but actually building things that function in the real world.

Projects may include examples such as:

- educational platforms
- mobile apps
- internal teacher/admin tools
- institutional websites
- automation systems
- experimental tools
- community/social platforms

---

## Technical Breadth / Stack Visibility

A secondary goal of the site is to preserve and present André’s wide technical background.

There should be a dedicated place where tools, frameworks, environments, and services can be documented clearly.

This could appear as:

- a dedicated stack page
- a section inside the homepage
- a section inside the about page
- or a combination of the above

Examples of items that may appear there include ecosystems such as:

- cPanel
- PHP
- WordPress
- Firebase
- Cloudflare
- Vercel
- Astro
- Next.js
- Flutter
- Capacitor
- React Native
- and many others

The structure should be flexible because the list is large and will evolve over time.

---

## About / Background Layer

Although the site is not a resume, it should still preserve a sense of professional background.

The “About” dimension of the site should explain that André’s experience comes from building, iterating, and maintaining real products and systems across different contexts.

This page or section should communicate:

- how broad the experience is
- that the work spans both product and technical implementation
- that the knowledge was built through real project execution
- that the background is multidisciplinary and practical

---

## Personality and Tone

The site should feel personal, but not informal in a careless way.

Tone should be:

- clear
- confident
- direct
- thoughtful
- builder-oriented
- practical

It should not sound like:

- recruiter-facing corporate copy
- startup buzzword overload
- exaggerated self-branding
- generic motivational language

---

## Navigation Philosophy

Navigation should be simple and intuitive.

The homepage should be highly visual and discovery-based, but the deeper structure should remain clean and usable.

A likely structure may include:

- Home
- Projects
- Project Detail Pages
- About
- Stack / Toolbox
- Contact

Optional future additions:
- Notes / Lab
- Experiments
- Archive
- Timeline

---

## Interaction Principles

### Fast browsing
The user should be able to understand the profile and browse projects quickly.

### Low friction
Clicking a project should first reveal enough information in a modal before deeper navigation is required.

### Visual clarity
The grid should remain readable and elegant even with multiple projects.

### Strong identity
The site should always feel like a personal digital profile, not an anonymous portfolio template.

---

## Visual System Principles

### Monochromatic foundation
The color system should be heavily restrained and likely monochromatic, with careful use of contrast.

### Social inspiration, professional execution
The site can intentionally evoke familiar social/profile UX patterns, but its final presentation should feel more curated and more premium.

### Image-first project discovery
Project exploration should be primarily visual.

### Tags for instant context
Project type tags are important because they immediately clarify what each item represents.

---

## Scalability

The structure of the website should support growth over time.

New projects should be easy to add without redesigning the homepage.

The system should support:

- many projects
- multiple categories
- bilingual content
- future experimental sections
- evolving technical stack documentation

This is important because the site is intended to grow alongside André’s body of work.

---

## What This Website Is Not

This project should **not** become:

- a job application website
- a plain resume page
- a random gallery with no context
- a literal Instagram clone
- a corporate agency site
- a landing page trying to sell one specific service

Its value comes from being a personal but highly structured public profile of a builder with many real-world projects.

---

## Summary

`gacitua.dev` should be a bilingual personal/professional website that presents André Gacitua as a builder of real digital products and systems.

The homepage should be strongly inspired by an Instagram profile structure, with:

- profile-style top section
- personal bio
- highlight-style navigation
- visual project grid
- project tags
- project modal previews
- deeper internal project pages

The visual language should be monochromatic, modern, and refined.

The final result should feel like a premium public profile for a developer/product builder — one that communicates breadth, execution, identity, and real experience without looking like a traditional portfolio or CV.
